Code Box

A code box is a component that acts as a container for CodeViews, allowing users to select and display different code samples. There are currently three types of code boxes, all of which extend a base class called CodeBox that defines some common methods and options.

Types of code boxes

There are currently following types of code boxes:

TabCodeBoxOffers ability to switch between code samples using tabs.
ProjectCodeBoxSimulates a complete project by displaying the entire source code.
VirtualCodeBoxCode box with no additional GUI, that is controlled only through code.

TabCodeBox

let x = 1;
let y = 2;

console.log("result: " + (x + y));
let string1 = "This is";
let string2 = " string.";

console.log(string1 + string2);

ProjectCodeBox

h1 {
    font-size: 16px;
    color: red;
}

let y = 2;

console.log("result: " + (x + y));

VirtualCodeBox

p {
    font-size: 1.6rem;
    line-height: 2.4rem;

    color: #E3EBEC;
}

Elements of code box

A code box is created from an element containing pre elements, which represent code views, and other elements that describe files. The following code sample demonstrates an example of such an element:

<!-- code box -->
<div id="MyCodeBox">
    <!-- code view -->
    <pre data-cb-name="main.js" data-cb-active><code>let x = 1;
let y = 2;

console.log("result: " + (x + y));</code></pre>

    <!-- downloadable file -->
    <div data-cb-name="Image.png" data-cb-file="./img/Image.png"></div>

    <!-- non-downloadable file -->
    <div data-cb-name="something.zip" data-cb-file></div>
</div>

Data attributes for code views

In addition to the code view options, code views within a code box can also include the following data attributes:

data-cb-nameSpecifies the name (identifier) of the code view. Each code view in the code box should have a unique name; duplicates will be ignored.
data-cb-activeMarks the code view as active. This code view will be automatically displayed when the code box is initialized.

Data attributes for files

Files in a code box are defined as elements with the data-cb-file attribute. Similar to code views, their name is set using the data-cb-name attribute. The following table describes these attributes:

data-cb-fileMarks the element as a file. A download link can be provided as its value. If no value is provided, the file will be non-downloadable.
data-cb-nameSpecifies the name (identifier) of the file. Each file in the code box should have a unique name; duplicates will be ignored.

Options

All code boxes inherit from the CodeBox class, so they also include the following options. Some of these options can also be configured using data attributes, with data attributes taking precedence.

lazyInitDetermines whether the code box should be initialized just before it is scrolled into the viewport.
implicitActiveSpecifies whether the first code view should be set as active by default if no other code view is set as active.
defaultCodeViewOptionsSpecifies default options for code views.
noCodeViewSelectedElementHeightSpecifies height of the element that is displayed when no code view is set as active.
noCodeViewSelectedTextSpecifies text of the element that is displayed when no code view is set as active.
minCodeViewLinesCountSpecifies the minimum number of lines in the code view that determines the minimal height of the code box when the code view is displayed.

lazyInit : boolean

Default value: true

Data attribute alternative: data-cb-lazy-init

Determines whether the code box should be initialized just before it is scrolled into the viewport. By default, this option is enabled (if disabled, the code box is initialized immediately). This helps reduce the load on performance during page loading.

implicitActive : boolean

Default value: false

Data attribute alternative: data-cb-implicit-active

Specifies whether the first code view should be set as active by default if no other code view is set as active.

defaultCodeViewOptions : CodeViewOptions

Default value: {}

Specifies default options for code views.

noCodeViewSelectedElementHeight : string

Default value: "50rem"

Data attribute alternative: data-cb-no-code-view-selected-element-height

Specifies height of the element that is displayed when no code view is set as active.

noCodeViewSelectedText : string

Default value: "No code view selected"

Data attribute alternative: data-cb-no-code-view-selected-text

Specifies text of the element that is displayed when no code view is set as active.

minCodeViewLinesCount : number

Default value: undefined

Data attribute alternative: data-cb-min-code-view-lines-count

Specifies the minimum number of lines in the code view that determines the minimal height of the code box when the code view is displayed. This value is not exact as it does not account for padding set by CSS styles; it is intended to set an optimal minimal height.
